Prints/send a single character to some opaque output entity

\begin{DoxyNote}{Note}
This function is not implemented by the library, only declared; you must provide an implementation if you wish to use the printf / vprintf function (and possibly for linking against the library, if your toolchain does not support discarding unused functions)

The output could be as simple as a wrapper for the {\ttfamily write()} system call on a Unix-\/like system, or even libc\textquotesingle{}s putchar , for replicating actual functionality of libc\textquotesingle{}s printf function; but on an embedded system it may involve interaction with a special output device, like a UART, etc.

in libc\textquotesingle{}s putchar, the parameter type is an int; this was intended to support the representation of either a proper character or EOF in a variable -\/ but this is really not meaningful to pass into putchar and is discouraged today. See further discussion in\+: \mbox{\hyperlink{}{c the single character to print }}
\end{DoxyNote}
